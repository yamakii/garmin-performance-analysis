# Data File Safety Rules

## CRITICAL: data/ と result/ 配下のファイルはgit未管理。削除したら復元不可能。

## 削除前チェック

データファイル（`data/`、`result/`、`$GARMIN_DATA_DIR`、`$GARMIN_RESULT_DIR` 配下）を削除する前に:

1. **中身を確認**: `ls -la` / `find ... -type f` でファイル一覧を表示
2. **ユーザーデータの有無を判断**: 空ディレクトリか、ユーザーが生成したファイルがあるか
3. **移動先を確認**: 誤った場所にあるファイルは、正しいパスに移動してから削除
4. **ユーザー確認**: ファイルが含まれる場合、削除前にユーザーに確認を取る

## 移動が必要なケース

- プロジェクトルート直下の `data/` `result/` → `$GARMIN_DATA_DIR` / `$GARMIN_RESULT_DIR` に移動
- パスバグで誤った場所に書かれたファイル → 正しいパスに移動後、誤ディレクトリを削除

## 禁止事項

- ファイル有無を確認せずに `rm -rf` でディレクトリごと削除
- git未管理ファイルを「復元可能」と仮定する
